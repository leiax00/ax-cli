# 架构总览

       │      │      │      │      │
       ▼      ▼      ▼      ▼      ▼
   ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
   │install│ │config│ │ env  │ │proxy │ │ info │
   │ cmd/ │ │ cmd/ │ │ cmd/ │ │ cmd/ │ │ cmd/ │
   └──┬───┘ └──┬───┘ └──┬───┘ └──────┘ └──────┘
      │        │        │
      ▼        ▼        ▼
   ┌─────────────────────────────┐
   │        commands/            │
   │  add / edit / rm / list / run│
   └──────────┬──────────────────┘
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
┌────────┐ ┌──────┐ ┌──────┐

## 分层设计

```
┌─────────────────────┐
│   CLI 层 (cli.rs)   │  clap 定义、参数解析
├─────────────────────┤
│  路由层 (main.rs)   │  命令分发
├─────────────────────┤
│  业务层 (commands/)  │  各命令的业务逻辑
├─────────────────────┤
│  基础设施层          │  config, detect, shell, tools, packages
├─────────────────────┤
│  外部依赖           │  git2, dirs, which, serde_yaml
└─────────────────────┘
```

## 数据流

### 配置加载流程

```
启动 → ConfigLoader::load()
         ├─ 检查 AX_CONFIG_DIR 环境变量
         ├─ 检查便携模式（二进制同级 config/）
         ├─ 回退到 ~/.config/ax-cli/
         ├─ 加载 config.yaml
         └─ 加载 config.d/*.yaml
```

### 命令执行流程

```
用户输入 → clap 解析 → Commands 枚举
                            │
                    main.rs match
                            │
                   commands/xxx::execute()
                            │
                    ┌───────┼───────┐
                    ▼       ▼       ▼
                 读取配置  执行操作  保存配置
                    │       │       │
                    └───────┴───────┘
                            │
                      git 自动同步（如已配置）
```

## 设计原则

1. **单一职责**：每个模块只做一件事
2. **幂等操作**：所有安装/部署操作可安全重复执行
3. **配置驱动**：行为通过 YAML 配置控制，减少硬编码
4. **零运行时依赖**：单一二进制，不依赖 Python/Node 等运行时
5. **渐进式复杂度**：简单场景一条命令搞定，复杂场景可逐步配置
